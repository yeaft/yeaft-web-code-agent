# 登录与注册

Yeaft 的访问入口是 Web 端的登录页。登录后才能看到 sidebar、Agent 列表、Workbench 等所有功能。

## 登录

1. 打开 server 的访问 URL（如 `https://yeaft.example.com`）
2. 输入**用户名**和**密码**
3. 如果启用了 TOTP（双因素认证）：
   - **首次登录**：页面显示一个 QR 码 — 用 Google Authenticator / Authy / 1Password 等扫码绑定，然后填 6 位验证码
   - **后续登录**：直接打开认证器应用，填当前 6 位码
4. 如果配置了邮箱验证，填写收到的验证码

> Server 的认证开关由部署时的 `ENABLE_AUTH` 决定。开发环境可以走 `skipAuth` 直接进。

## 注册

公开注册**默认关闭** — 必须用管理员发的邀请码注册：

1. 登录页点 **"注册新账号"**
2. 填写：
   - **用户名**（≥2 字符）
   - **密码**（≥6 字符）
   - **确认密码**
   - **邮箱**（可选 — 配了邮件发送会用来发验证邮件 / 找回密码）
   - **邀请码**（管理员在 设置 → 邀请码管理 创建）
3. 点 **提交** 完成注册
4. 注册成功后自动跳到登录页

> 邀请码可一次性、可多次（管理员可配），有效期可设。已使用 / 已过期的邀请码不能再注册。

## 修改密码

进入 **设置 → 安全** → 修改密码：

1. 输入当前密码
2. 输入新密码（≥6 字符）
3. 再次确认新密码
4. 点 **修改密码**

修改成功后**当前 session 不会自动登出**，但其他设备需要用新密码重新登录。

## 找回密码

如果配置了邮箱发送：

1. 登录页点 **忘记密码**
2. 输入注册时的邮箱
3. 收邮件，点链接，设新密码

如果没有配邮箱或忘了邮箱，**联系管理员**在后台手动重置（admin 在 dashboard 里有重置接口）。

## 多账号 / 角色

- **Pro** — 普通用户角色，可以新建会话、用所有 backend、管自己的 Agent
- **Admin** — 管理员，能开邀请码、看 dashboard、管全局配置

管理员账号通常是第一个创建的；后续账号通过邀请码加入。账号角色在创建邀请码时指定。

## 登出

设置 → 账户 → **退出登录**。

登出后 token 失效，需要重新登录。
