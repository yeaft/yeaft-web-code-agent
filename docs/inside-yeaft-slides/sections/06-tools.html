06 · Action surface

Tools, Skills, and MCP are different layersTools、Skills、MCP 是三层不同能力

Tools

Built-in executable primitives

内置可执行原语

File read/write/edit, patch, shell, search, web, notebooks, tasks, sub-agents, planning, image viewing, and more. They are registered in Yeaft's ToolRegistry and filtered by mode/config.

文件读写编辑、patch、shell、搜索、Web、notebook、后台任务、子 Agent、计划、图片查看等。它们注册在 Yeaft ToolRegistry 中，并按 mode/config 过滤。

Skills

Reusable workflows

可复用工作流

Local markdown skills under ~/.yeaft/skills/ package domain workflows, references, and templates. The Skill tool lists, searches, and loads them on demand.

本地 ~/.yeaft/skills/ 下的 markdown skill 封装领域工作流、参考资料和模板。Skill 工具可以按需列出、搜索和加载。

MCP

External tool bridges

外部工具桥接

MCP servers extend the action surface without baking every integration into Yeaft. The MCP manager connects servers and exposes their tools through the same call loop.

MCP server 扩展行动面，不需要把每个外部集成都写死进 Yeaft。MCP manager 连接 server，并把工具暴露到同一套调用循环里。

Integration pattern: advertise capability → LLM chooses action → registry/manager executes → raw result is persisted → folded result returns to context.

集成模式：声明能力 → LLM 选择动作 → registry/manager 执行 → 原始结果持久化 → 折叠后的结果回到上下文。
