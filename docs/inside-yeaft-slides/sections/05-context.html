05 · Prompt / memory / dream

Context is assembled, not dumped上下文是组装出来的，不是倾倒进去的

Prompt

Templates + persona

模板 + 人格

Base identity, Yeaft rules, mode instructions, tool guidance, runtime platform, and VP persona are composed before the call.

基础身份、Yeaft 规则、mode 指令、工具指南、运行平台和 VP 人格会在请求前组合。

Memory

Scope-aware recall

按 scope 召回

H2-AMS separates user, VP, sub-agent, Session, feature, and global scopes, then injects relevant recall instead of full transcripts.

H2-AMS 区分 user、VP、sub-agent、Session、feature、global 等 scope，只注入相关召回，而不是塞完整 transcript。

Dream

Background consolidation

后台巩固

Dream turns volatile interaction diffs into durable atomic memory segments and summary layers.

Dream 把临时交互 diff 整理成可持久化的原子记忆段和 summary 层。

Yeaft's goal is controlled context growth: preserve what matters, expose what was injected, and compact deliberately.

Yeaft 的目标是可控的上下文增长：保留重要信息，暴露注入内容，并有意识地 compact。
