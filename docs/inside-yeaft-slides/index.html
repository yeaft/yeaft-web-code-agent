First framing

Yeaft has two surfaces.

Yeaft 有两大类能力。

Provider surface: run external code-agent providers such as Claude Code CLI or GitHub Copilot CLI behind the same Web UI.

Provider 表面：把 Claude Code CLI、GitHub Copilot CLI 等外部 code-agent provider 接到同一套 Web UI。

Native engine: Yeaft's own Session-based code-agent engine with VP, memory, tools, multi-provider routing, and inspectable execution.

原生引擎：Yeaft 自己的 Session 型 Code Agent engine，包含 VP、记忆、工具、多 provider 路由和可观测执行细节。

This deck focuses on the native engine, because it exposes the architecture instead of hiding it behind a provider.

这套 slides 会主要讲后者，因为原生引擎把架构暴露出来，而不是把一切藏在 provider 后面。
